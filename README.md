# Copycat Editor — 構造理解型コード写経ツール

## 概要

他人のコードを貼り付けると、AIがモジュール・構造の単位に自動分解し、複数ステップで段階的に模写できるWebアプリ。写経をただの作業ではなく、仕組みを理解しながら真似できる機会へ昇華させる。

[[projects/walking-coding/README|walking coding]]の一環として開発。

## 背景・動機

- walking codingで「他人のコードを写経する」習慣があるが、芸術系コードは `a`, `b`, `t` のような変数名が多く読みにくい
- コード全体を一度に写経すると、構造が見えないまま「打つ作業」になりがち
- 構造を分解してステップごとに模写することで、パーツの役割を理解しながら手を動かせる
- [[research/nare-to-tae/README|「慣れと耐え」]]のRQとも接続 — 難解なコードに向き合う「耐え」を支援するツール

## 仕様（v1）

### ユーザーフロー

1. **コード入力** — 左ペインに「ここにコードをペースト」。コードを貼り付けて「開始」ボタン押下
2. **解析** — 「コードを解析中...」表示。AIがコードを構造解析し、ステップに分解
3. **ステップ式写経** — 完成までのステップ数が表示される。各ステップに：
   - 写経すべきコード（差分ハイライト付き。git diffのように追加部分が別色）
   - 短い解説文：「まずは...というモジュールから作っていきます。これは...という役割をしています。」
4. **段階的積み上げ** — ステップ2以降は「前ステップのコード + 新規コード」の構成。差分が視覚的にわかる
5. **最終ステップ** — 完全版のコードが組み上がる

### 分解のロジック

- **基盤から積み上げる方式。** 最低限の機能を持つコードを最初に書かせ、徐々に磨いていく
- 例：「歪んだ円」のプログラム → ステップ1: 極座標系で正円を描画 → ステップ2: 歪みを追加 → 段階的に複雑なルックへ
- 基本は関数単位だが、意味的にわかりやすい粒度を優先

### 変数リネーム

- 可読性の低い変数名（`a`, `b`, `t`）にコメントで意味を付記：`t = 0.1 // t = time`
- 完全リネームすると冗長になるため、コメント方式を採用
- 元のコードとの対応関係が失われないようにする

### ステップ粒度の動的調整

- 各ステップ横にボタン：「もっと細かく」「もっとまとめて」
- 粒度が大きすぎる/小さすぎると感じたら、写経中に調整できる

### エラーアシスト

- 写経の完全一致判定は不要
- ロジック的な差異（実行時エラーになるもの）はチェック
- エラーを吐いた場合にアシストを表示

### 脱線の許容

- 各ステップの解説文に「写し終わったら、変数tの値を変えてみましょう。」等の呼びかけ
- 将来的にはlil-gui的なパラメータ操作UIを検討（v1では見送り）

### プレビュー実行

- 左右のペインそれぞれに「実行」ボタン
- 実行結果は画面にオーバーレイでフルスクリーン表示
- **対応：p5.js / WebGL**（v1）。他のコードは後回し

### UI構成

```
┌─────────────────────┬─────────────────────┐
│  左ペイン            │  右ペイン            │
│  元コード（ステップ表示） │  写経スペース         │
│  解説文              │                     │
│  [▶ 実行]            │  [▶ 実行]            │
│                     │                     │
│  [◀ 前へ] Step 2/8 [次へ ▶] │              │
│  [もっと細かく] [まとめて]  │                │
└─────────────────────┴─────────────────────┘
```

## 技術スタック

| 要素 | 技術 |
|---|---|
| フロントエンド | Web app（フレームワーク未定。React or Vanilla） |
| コードエディタ | **CodeMirror** |
| AI解析 | Claude API / Gemini API（切り替え可能な抽象化） |
| APIキー | **BYOKey方式。** ユーザーが自分のAPIキーをSettingsから入力。localStorageに保存 |
| プレビュー | p5.js / WebGL をiframeで実行 |

## プロトタイプの最小スコープ

1. コード貼り付け → AI解析 → ステップ分解（解説文付き）
2. 左右分割画面（CodeMirror × 2）
3. ステップ送り（前へ/次へ）、差分ハイライト
4. p5.jsのプレビュー実行
